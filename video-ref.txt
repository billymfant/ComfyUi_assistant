Here's a breakdown of what Claude Code will likely find and what to expect:
What Your Setup Probably Looks Like
LTX 2.3 outputs video as batched IMAGE tensors — shape (frames, H, W, 3) — so your custom nodes slot directly into the pipeline without needing a separate video type. That's the good news; you're working with standard ComfyUI IMAGE format throughout.

The Two Nodes — What They Actually Need to Do
TimeSliceGrid Node
The core idea is: divide each frame into an NxN grid, then for each tile, pull its content from a different point in time rather than the current frame. The offset per tile creates the mosaic/shatter effect.
Frame 10 displayed, but:
  Tile [0,0] → pulled from frame 8
  Tile [0,1] → pulled from frame 11
  Tile [1,0] → pulled from frame 7
  Tile [1,1] → pulled from frame 13
The stagger pattern (random, radial, linear) controls the aesthetic. This is computationally light — it's just tensor indexing, no model inference.
Inputs you'll want:

images (IMAGE) — full frame batch from LTX
grid_size (INT) — NxN, probably 4–16
time_offset_range (INT) — how many frames ahead/behind to pull from
stagger_pattern (STRING) — random, radial, linear
seed (INT) — for reproducible randomness


GlitchOverlay Node
This composites procedural graphic elements onto frames — colored rectangles, thin bars, pseudo-code text fragments. The key is making them feel like data artifacts, not decorations. They should flicker — appearing on some frames and not others — with slight positional drift.
Inputs you'll want:

images (IMAGE)
element_count (INT) — how many artifacts per frame
flicker_rate (FLOAT) — probability an element appears on any given frame
color_palette (STRING) — neon, monochrome, data
seed (INT)


Architectural Risks Claude Code Will Likely Flag
Tensor boundary clamping — when a tile tries to pull from frame -2 or frame N+5, you need to clamp or mirror. Easy fix but easy to miss initially.
LTX frame count — LTX 2.3 generates fixed-length clips. If your grid offset range exceeds the clip length, tiles will hit empty indices. Claude Code should catch this once it reads the LTX node outputs.
Batch vs single frame — some nodes in ComfyUI pass single frames (1, H, W, 3) rather than the full batch. Your nodes need to handle both or explicitly document which they expect.
Text rendering for overlay — PIL/Pillow is available in ComfyUI's environment for drawing text and shapes. You'll need to convert tensors → PIL → back to tensor for the overlay compositing step. Claude Code should scaffold that conversion.
