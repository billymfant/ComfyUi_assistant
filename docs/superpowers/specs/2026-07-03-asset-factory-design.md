# Asset Factory — design spec (2026-07-03)

**Direction chosen in brainstorm:** ComfyUI is the asset *builder*; the user assembles and
animates scenes by hand in Blender. AI look-dev/finishing on renders (A), a Blender addon
bridge (B) and a rigged character pipeline (D) are explicitly parked until this ships.

## Goal

One master pipeline that goes **concept → 4 orthographic views → textured 3D asset**, by
combining the two proven pieces: the ortho multiview generator (wf 10, MV-Adapter) and the
Trellis 2 GGUF mesh generator.

User flow ("press play" UX):

1. Iterate the concept image in the CONCEPT group (Flux 2 t2i, optional image reference) —
   seconds per try, 3D groups bypassed.
2. When the concept is right, enable the MULTIVIEW + TRELLIS groups (Ctrl+B) and queue.
   ComfyUI's output caching reuses the unchanged concept (same seed), so the 3D stages run
   on the approved image without regenerating it.
3. Output: `asset.glb` (UVs + baked PBR texture) + the 4 view images (reference sheet kept
   as a deliverable — assembled turnaround comes with the phase-2 builder).

## Quality bar (user-confirmed)

**Clean-up-friendly tier:** ~200k faces, holes filled, floaters/inner faces removed, Xatlas
UVs, 2048 texture. 15–30 min of Blender polish per asset, not remodeling. Covers characters,
props and environment set pieces alike; texturing happens inside generation (no separate
texturing tool for hand-made meshes — descoped).

## Deliverables

1. **wf 12_TRELLIS_TEXTURED** (single image → textured mesh) — the open task from HANDOFF.
   Chain: LoadImageWithTransparency → PreProcessImage (slot 2) + LoadModel (GGUF Q4_K_M,
   flash_attn) → MeshWithVoxelAdvancedGenerator → Remesh → SimplifyMesh (200k) →
   FillHolesWithMeshlib → MeshWithVoxelToTrimesh (90°) → MeshTexturing (Xatlas) →
   ExportMesh glb. Generated by `builders/gen_trellis_textured.py`.
2. **wf 13_ASSET_FACTORY** (the master) — groups G1 CONCEPT (Flux 2, on) / G2 MULTIVIEW
   (MV-Adapter i2mv SDXL, front/right/back/left, BiRefNet `ZhengPeng7/BiRefNet`, bypassed) /
   G3 TRELLIS 3D (MultiViewGenerator 1024_cascade → PostProcessAndUnWrapAndRasterizer
   [200k faces, Xatlas, texture 2048] → ExportMesh + Preview3D, bypassed). Generated by
   `builders/gen_asset_factory.py`.
3. **`builders/make_asset.py`** (phase 2, after UI proven): one command → organized
   `output/AssetFactory_<Name>/` package with glb, textures, labeled sheet, views, concept.

## Key technical facts (verified live this session)

- `Trellis2MeshWithVoxelMultiViewGenerator_GGUF` accepts exactly **4 views**: front
  (required) + back/left/right (optional). No top/bottom input.
- `Trellis2PostProcessAndUnWrapAndRasterizer_GGUF` is an all-in-one tail (remesh, simplify,
  fill holes, remove floaters/inner faces, Xatlas unwrap, texture bake → trimesh + base
  color + metallic/roughness) — the multiview chain is only 3 nodes after preprocessing.
- The GGUF pack's example workflows have stale widget layouts (crash cause last session);
  all workflows here are generated with `wf_lib` against live `/object_info` widget order.
- Only `uv_unwrap_method=Xatlas` works on this machine (`Smart` needs the smart_uv wheel,
  `Blender` needs bpy).

## Risks

- **VRAM (16 GB):** Flux2 + SDXL/MV-Adapter + Trellis in one graph relies on ComfyUI model
  unloading; if OOM, add VRAM-purge nodes between groups or split G3 into its own run.
- Trellis geometry stays "clean-up-friendly", not hero-topology; that is the accepted bar.
